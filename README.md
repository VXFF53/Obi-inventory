Project Overview
Purpose

Obi Inventory was the initial prototype created to explore the fundamentals of an inventory system.

It focused on:

Displaying product data
Basic UI structure
Understanding frontend workflows
Key Features
Static product listing
Basic layout structure
Simple UI components
🧱 Technical Stack
HTML
CSS
JavaScript
Vercel (deployment)
⚠️ Limitations
No real data management
No CRUD functionality
No backend/data persistence
No analytics or insights
No system architecture
❌ Mistakes
1. Built UI Without System Design
No planning for:
data flow
scalability
architecture
2. Too Basic for Real Use
Did not reflect real inventory workflows
🧠 Key Learning

“A UI alone is not a system.”

This version highlighted the need for:

real functionality
structured data handling
better planning
📊 Verdict

❌ Not portfolio-ready
✅ Important foundational step
